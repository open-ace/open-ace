# 页面刷新定制化功能实现总结

## 实现内容

本次实现完成了页面刷新定制化功能，移除了 Header 中的全局刷新按钮，为各页面提供了独立的刷新控制能力。

## 核心架构

### 1. 状态管理层（Zustand Store）
- **文件**: `frontend/src/store/pageRefreshStore.ts`
- **功能**:
  - Per-page refresh configuration
  - localStorage 持久化（版本化存储）
  - 多标签页同步机制
  - 自动清理过期配置

### 2. Hook 层
- **usePageRefresh** (`frontend/src/hooks/usePageRefresh.ts`)
  - 查询键过滤刷新逻辑
  - 请求去重
  - 刷新节流/防抖
  - 并发刷新冲突处理
  - 错误处理和降级策略

- **useGlobalRefreshPause** (`frontend/src/hooks/useGlobalRefreshPause.ts`)
  - 全局暂停/恢复功能
  - 键盘快捷键支持（Ctrl+Shift+P）

### 3. 组件层
- **PageRefreshControl** (`frontend/src/components/common/PageRefreshControl.tsx`)
  - 自动刷新开关
  - 间隔选择器
  - 手动刷新按钮
  - 刷新状态指示器
  - 错误指示器
  - 紧凑模式（移动端适配）

### 4. 查询键管理
- **queryKeyRegistry** (`frontend/src/utils/queryKeyRegistry.ts`)
  - 查询键命名规范
  - 注册机制
  - 刷新范围管理
  - 已注册所有页面的查询键

- **queryKeyMatcher** (`frontend/src/utils/queryKeyMatcher.ts`)
  - 精确匹配
  - 前缀匹配
  - 排除规则

## 关键特性

1. **查询键精确过滤**: 避免刷新范围过广，不刷新 iframe 内嵌数据
2. **请求去重**: 同一查询键在 1秒时间窗口内不重复刷新
3. **刷新节流**: 手动刷新按钮点击后禁用 1-2秒
4. **错误处理**: 刷新失败显示错误指示器，超过最大重试次数后降级
5. **状态持久化**: localStorage 持久化用户偏好，跨页面保持
6. **多标签页同步**: 监听 storage 事件，实时同步状态
7. **全局暂停**: 快捷键 Ctrl+Shift+P 暂停/恢复所有自动刷新
