## 一、Open ACE 是什么？

**Open ACE** 是一个开源的**企业级 AI 工作平台**，帮你解决两大问题：

| 问题 | 解决方案 |
|------|----------|
| 🤔 **AI 用得好不好？** | **Work 模式** — 让你高效使用 AI，提升工作效率 |
| 😰 **AI 管得住管不住？** | **Manage 模式** — 让管理者全面掌控 AI 使用情况，降低风险 |

**核心能力：**
- 🤖 **多 AI 工具集成** — 支持 Qwen Code、Claude Code、OpenClaw 等主流 AI，一个平台全搞定
- 💬 **智能对话管理** — 历史记录、会话恢复、上下文记忆，对话不中断
- 📝 **提示词库** — 团队共享优质提示词，最佳实践一键复用
- 🌐 **远程工作区** — 在浏览器中操作远程机器，AI 直接在远程运行
- 📊 **用量分析** — Token 消耗、成本分析、效率提升，用数据说话

**适合谁用？**
- 需要频繁使用 AI 编码助手的开发者
- 希望统一管理 AI 工具和成本的团队
- 想要私有化部署、保障数据安全的组织

## 二、首次使用前的准备

### 1. 基础环境

在开始之前，请确保：

- ✅ **Open ACE 已部署** — 你的团队已经部署好 Open ACE 平台（本地或远程）
- ✅ **已注册账号** — 你拥有登录账号（联系管理员获取，或使用默认账号）
- ✅ **浏览器支持** — 推荐使用 Chrome、Edge、Firefox 等现代浏览器

### 2. 登录平台

1. 打开浏览器，访问 Open ACE 地址（如 `http://localhost:5000` 或你的部署地址）
2. 输入**用户名**和**密码**，点击登录

> 💡 **默认管理员账号**：`admin` / `admin123`（生产环境请务必修改默认密码！）

### 3. 关键配置项（管理员操作）

如果你是**管理员**，首次使用前需要配置：

#### 添加 API Key

**为什么需要配置？** Open ACE 本身不提供 AI 模型，它帮你**统一管理**多个 AI 工具的 API Key，让团队成员无需各自配置。

**配置入口：**
1. 登录后，切换到 **Manage 模式**（点击顶部模式切换器）
2. 左侧导航栏 → **远程工作区** → **API 密钥**
3. 点击 **"添加 API 密钥"** 按钮
4. 选择 Provider（如 OpenAI、Anthropic、Google），输入 API Key 和可选的 Base URL
5. 点击保存

> ⚠️ **重要**：远程工作区功能**必须**在管理页面添加 API Key，环境变量中的 Key 不会自动生效。

## 三、核心功能快速上手

### 场景 1：如何新建一个对话会话

1. 登录后，默认进入 **Work 模式**（工作模式）
2. 在**左侧边栏**，找到 **"会话"** 区域
3. 点击 **"新建会话"** 按钮（通常是一个 `+` 图标）
4. 在弹出的对话框中：
   - 选择**工作区类型**（本地工作区 或 远程工作区）
   - 如果选择远程工作区，从机器列表中选择一台**在线的远程机器**
   - 输入**项目路径**（远程机器上的工作目录，默认自动填充）
   - 输入**会话标题**（可选，方便后续查找）
5. 点击 **"创建"**，新的工作区标签页即会打开

### 场景 2：如何选择不同的 AI 模型

1. 在工作区页面，找到顶部的**工具选择器**
2. 点击下拉菜单，你可以看到可用的 AI 工具：
   - **Qwen** — 通义千问系列（如 qwen3-coder-plus）
   - **Claude** — Anthropic Claude 系列
   - **OpenClaw** — 其他 OpenAI 兼容模型
3. 选择你想要的工具后，系统会自动加载对应的模型
4. 部分工具支持**指定模型名称**，在创建会话时输入即可

> 💡 **提示**：可用的 AI 工具取决于管理员在后台配置的 API Key。如果某个工具无法使用，请联系管理员确认 API Key 是否有效。

### 场景 3：如何使用提示词模板

**什么是提示词模板？** 提示词模板是预先写好的优质提示词，帮你快速开始，不用每次都从零写提示词。

**使用方法：**

1. 在**右侧边栏**，点击 **"提示词"** 标签页
2. 你会看到一个提示词列表，点击任意提示词可以查看详情
3. 点击提示词旁边的 **复制图标**，即可将内容复制到剪贴板
4. 将复制的内容粘贴到工作区的输入框中，按需修改后发送

**浏览更多提示词：**
1. 点击左侧边栏的 **"提示词"** 导航项
2. 进入提示词管理页面，可以浏览、搜索、筛选所有提示词
3. 点击任意提示词卡片，查看完整内容和使用说明

## 四、界面功能说明

Open ACE 提供两种模式：**Work 模式**（所有用户可用）和 **Manage 模式**（仅管理员可用）。

### 🔵 Work 模式 — 所有用户都可以访问

Work 模式是普通用户的日常工作界面，采用**三栏布局**：

```
┌─────────────────────────────────────────────────────────────┐
│  [Logo] Open ACE        [模式切换器]        [用户信息]      │  ← 顶部导航栏
├──────────┬──────────────────────────────┬──────────────────┤
│          │                              │                  │
│  会话    │                              │  提示词          │
│  列表    │      主工作区                │  工具            │  ← 右侧工具栏
│          │   （对话/代码操作）           │  文档            │
│          │                              │                  │
│  导航    │                              │                  │
│  菜单    │                              │                  │
├──────────┴──────────────────────────────┴──────────────────┤
│  [状态栏：Token 用量 | Request 用量]                         │  ← 底部状态栏
└─────────────────────────────────────────────────────────────┘
   ↑ 左侧边栏              ↑ 中间主界面
```

#### 左侧导航栏

| 导航项 | 功能说明 |
|--------|----------|
| **Workspace（工作区）** | 默认的 AI 对话界面，在这里与 AI 交互、编写代码 |
| **Sessions（会话）** | 查看所有历史会话，支持搜索和筛选，可恢复历史对话 |
| **Prompts（提示词）** | 浏览和管理提示词库，快速复用优质提示词 |
| **My Usage（我的用量）** | 查看个人的 Token 使用情况和趋势分析 |
| **Insights（洞察）** | 获取 AI 使用效率分析和优化建议 |

#### 中间主界面

- **对话区域**：显示你与 AI 的对话历史，包括消息、代码块、思考过程等
- **输入框**：在底部输入你的问题或指令，按 Enter 发送
- **工具栏**：输入框上方或侧边，提供发送、暂停、停止等操作按钮
- **全屏模式**：点击全屏按钮，隐藏左右面板，专注对话（按 ESC 退出）

#### 右侧工具栏

| 标签页 | 功能说明 |
|--------|----------|
| **提示词** | 快速浏览常用提示词，一键复制 |
| **工具** | 快捷入口，切换不同的 AI 工具（Qwen、Claude、OpenClaw） |
| **文档** | 帮助文档入口（如 Getting Started、FAQ 等） |

> 💡 **提示**：右侧面板可以折叠/展开，点击面板标题旁的箭头即可切换。

## 五、常见入门问题与排查（FAQ）

### Q1：为什么发送消息没有响应？

**可能原因和解决方法：**

1. **API Key 未配置或无效**
   - 联系管理员确认 API Key 是否已正确配置
   - 检查 API Key 是否过期或被禁用

2. **网络连接问题**
   - 检查浏览器网络是否正常
   - 尝试刷新页面或重新登录

3. **AI 工具未选择**
   - 确认顶部工具选择器中已选择有效的 AI 工具
   - 如果工具列表为空，说明后台未配置可用的 API Key

### Q2：模型无法选择/报错怎么办？

**常见情况：**

- **工具显示但无法使用** → API Key 可能无效，联系管理员检查
- **模型名称报错** → 确认输入的模型名称正确（如 `qwen3-coder-plus`）
- **提示"无可用 API Key"** → 管理员需在管理页面添加对应的 API Key

### Q3：如何切换不同的对话模式？

Open ACE 有两种模式：

| 模式 | 入口 | 适用人群 |
|------|------|----------|
| **Work 模式** | 默认模式，所有用户可用 | 普通用户、开发者 |
| **Manage 模式** | 点击顶部模式切换器 → 选择"Manage" | 仅管理员可见 |

**切换方法：**
1. 在页面顶部，找到 **模式切换器**（通常显示"Work"或"Manage"）
2. 点击切换器，选择你想要的模式
3. 页面会自动跳转到对应模式的首页

### Q4：如何找回之前的对话？

1. 点击左侧边栏的 **"Sessions（会话）"**
2. 使用**搜索框**输入关键词，筛选历史会话
3. 点击任意会话卡片，即可查看完整对话
4. 如果想继续该会话，点击 **"恢复会话"** 按钮

### Q5：远程工作区看不到机器怎么办？

**排查步骤：**

1. **确认机器已注册** → 联系管理员确认远程机器是否已注册并在线
2. **确认已分配权限** → 管理员需在管理页面将机器分配给你
3. **检查网络连通性** → 远程机器需能访问 Open ACE 服务器
4. **查看机器状态** → 如果机器显示"离线"，需检查远程 Agent 是否正常运行

### Q6：如何复制提示词？

1. 在右侧面板的"提示词"标签页，找到你想要的提示词
2. 点击提示词右侧的 **复制图标**（剪贴板图标）
3. 系统会提示"已复制"，然后粘贴到工作区输入框即可

### Q7：Token 用量在哪里查看？

- **个人用量**：左侧边栏 → **My Usage（我的用量）**
- **详细报告**：左侧边栏 → **Insights（洞察）**
- **实时用量**：底部状态栏会显示当前会话的 Token 消耗

### Q8：如何退出全屏模式？

- 按键盘上的 **ESC** 键
- 或者点击全屏按钮（如果可见）
