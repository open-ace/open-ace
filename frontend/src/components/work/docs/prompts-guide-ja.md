## プロンプトとは

**プロンプト（Prompt）** は、AI に送信する指示や質問です。簡単に言えば、プロンプトは AI とのコミュニケーション言語です。

### プロンプト例

```
次週のプロジェクト進捗報告に関する会議招待メールを書いてください。
```

良いプロンプトは：
- ✅ AI にニーズを正確に理解させる
- ✅ 高品質な回答を得る
- ✅ コミュニケーション時間を節約し、効率を向上

## プロンプトの基礎

### 1. プロンプトの基本構造

完全なプロンプトは通常、以下の要素を含みます：

| 要素 | 説明 | 例 |
|------|------|------|
| **役割設定** | AI に役割を指示 | "あなたはシニアプロダクトマネージャーです" |
| **タスク説明** | 何を完了するか明確に | "製品要求文書を書いてください" |
| **背景情報** | 必要なコンテキストを提供 | "これは中小企業向けのコラボレーションツールです" |
| **出力要件** | 期待する出力形式を指定 | "Markdown 形式を使用、ユーザーストーリーと受け入れ基準を含む" |

### 2. プロンプトのタイプ

#### 会話型プロンプト
日常の Q&A とインタラクション用：

```
アジャイル開発とは何ですか？簡単な言葉で説明してください。
```

#### タスク型プロンプト
具体的な作業を完了用：

```
以下の英語メールを日本語に翻訳し、ビジネス調を維持してください：

Dear Team,
I would like to schedule a meeting...
```

#### クリエイティブ型プロンプト
クリエイティブなコンテンツ生成用：

```
若者向けスポーツドリンクのブランドキャッチコピーを設計してください：
- 短く力強く、8文字以内
- 活力とスポーツ精神を反映
- 簡単に覚えられる
```

### 🔍 スマート検索

セッション間で会話履歴を検索、知識を蓄積：
- キーワードで検索
- 時間範囲でフィルタ
- ツールタイプでフィルタ

## 効果的なプロンプト作成技法

### 技法 1：明確な役割設定

❌ 悪い例：
```
製品説明を書いてください
```

✅ 良い例：
```
あなたは10年の経験を持つ技術文書作成の専門家です。
新開発の CRM システムの製品マニュアルを書いてください：
1. 製品概要
2. コア機能紹介
3. クイックスタートガイド
4. FAQ

要件：簡潔で専門的な言葉を使用、過度な技術用語を避ける
```

### 技法 2：十分な背景を提供

❌ 悪い例：
```
売上データを分析してください
```

✅ 良い例：
```
2024年第1四半期の売上データを分析する必要があります。

背景情報：
- 製品：企業向け SaaS ソフトウェア
- 目標市場：アジア太平洋地域の中小企業
- データ範囲：2024年1月〜3月
- 分析目的：市場拡大戦略の効果を評価

以下の観点から分析してください：
1. 売上傾向
2. 顧客獲得コスト
3. 地域分布
4. 製品ライン貢献度

出力形式：表とグラフで主要データを表示
```

### 技法 3：構造化された出力要件

特定の形式が必要な場合、明確に指定：

```
製品情報を以下の JSON 形式で出力してください：

{
  "product_name": "製品名",
  "features": ["機能1", "機能2"],
  "target_users": "対象ユーザー",
  "pricing": "価格戦略"
}

オンライン教育プラットフォームの製品情報を生成してください。
```

### 技法 4：段階的なガイド

複雑なタスクの場合、段階的に AI をガイド：

```
新製品のマーケティング計画を立ててください：

ステップ1：ターゲットユーザー層を分析
ステップ2：コアセールスポイントを決定
ステップ3：マーケティングチャネルを選択
ステップ4：コンテンツ戦略を策定
ステップ5：KPI 目標を設定

まずステップ1を完了し、確認後に続けてください。
```

### 技法 5：例を使用してガイド

例を使用して AI にニーズを理解させる：

```
以下のスタイルで製品宣伝文案を書いてください：

例：「複雑さを告别し、効率を拥抱する — XX コラボレーションプラットフォーム」

スマートカスタマーサービスシステムの宣伝文案を書いてください。
製品特徴：
- 24時間オンラインサポート
- 多言語サポート
- スマート学習最適化
```

## ベストプラクティス

### ✅ DO - 推奨事項

1. **明確で具体的**：ニーズを具体的に表現
2. **コンテキストを提供**：十分な背景情報を与える
3. **反復最適化**：AI の回答に基づいてプロンプトを調整
4. **テンプレートを保存**：効果的なプロンプトをライブラリに保存
5. **プライバシー保護**：プロンプトに機密情報を含めない

### ❌ DON'T - 避けるべき事項

1. **曖昧**：「何か書いて」のような曖昧な指示を避ける
2. **情報過多**：一度に過多の情報を提供
3. **コンテキスト無視**：会話中の前の交流を無視
4. **過度な期待**：AI の限界を理解する
5. **秘密漏洩**：プラットフォーム外で機密情報を共有しない

## よくある質問

### Q1: AI の回答が期待したものではない理由？

**可能な原因：**
- プロンプトが明確で具体的でない
- 必要な背景情報が不足
- 出力要件が具体的でない

**解決方法：**
- プロンプトを再構成し、具体的要件を追加
- 期待する出力形式の例を提供
- AI を段階的にガイド

### Q2: AI に前の会話内容を記憶させる方法？

Open ACE はセッションコンテキストを自動保存：
- 同じセッションで会話を続ける
- 「履歴セッション」機能で前のセッションを復元

### Q3: プロンプトライブラリのテンプレートの使用方法？

1. 「プロンプトライブラリ」を開く
2. 必要なテンプレートを見つける
3. 「使用」をクリック
4. プレースホルダー内容を変更
5. AI に送信

### Q4: チームメンバーとプロンプトを共有する方法？

1. プロンプトを作成または編集
2. 「共有範囲」を選択
3. 「チーム」または「全社」を選択
4. 保存をクリック

### Q5: AI の回答が長すぎ/短すぎる場合？

プロンプトで明確に指定：
- 簡潔な回答が必要：「3文以内で回答してください」
-詳細な回答が必要：「例を含めて詳細に説明してください」
