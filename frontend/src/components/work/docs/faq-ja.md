## 一、インストールと展開

### Docker起動失敗：データベース接続タイムアウト

**現象：** Dockerコンテナ起動時、ログに表示：`ERROR: PostgreSQL not ready after 60s. Exiting.`

**原因：**
1. PostgreSQLコンテナが初期化未完了
2. データベース接続パラメータ設定エラー
3. Dockerネットワーク問題によりコンテナ間通信不可

**解決手順：**
1. PostgreSQLコンテナ状態確認：`docker compose ps`
2. PostgreSQLログ確認：`docker compose logs postgres`
3. PostgreSQLが初期化中の場合、完了待機後再起動：`docker compose restart open-ace-web`
4. データベース接続パラメータ確認

**予防策：** docker-compose.ymlにdepends_onとhealthcheckを設定

---

### ポート競合：5000ポートが使用中

**現象：** 起動時エラー：`Error: Address already in use (0.0.0.0:5000)`

**原因：**
1. 他のサービスが5000ポート使用中
2. 前回のOpen ACEプロセスが完全停止していない

**解決手順：**
1. ポート使用状況確認：`lsof -i :5000`または`netstat -tlnp | grep 5000`
2. ポート使用プロセス停止：`kill -9 <PID>`または`docker compose down`
3. 別ポートで起動：`PORT=8080 docker compose up -d`

---

### SECRET_KEY未設定：本番環境起動失敗

**現象：** コンテナ起動失敗、ログに表示：`RuntimeError: SECRET_KEY environment variable must be set in production!`

**原因：**
1. 本番環境でSECRET_KEY環境変数未設定
2. デフォルト開発キー使用

**解決手順：**
1. SECRET_KEY環境変数設定：`echo "SECRET_KEY=$(openssl rand -hex 32)" > .env`
2. docker-compose.ymlで環境変数設定
3. コンテナ再起動

---

### 設定ファイルが見つからない

**現象：** 起動後Workspace機能使用不可、ログに表示：`Config file not found: ~/.open-ace/config.json`

**解決手順：**
1. 設定ディレクトリとファイル作成：`mkdir -p ~/.open-ace`し、サンプル設定コピー
2. 設定ファイル編集でhost_name等パラメータ変更
3. サービス再起動

---

## 二、ログインと認証

### ログイン失敗：ユーザー名またはパスワードエラー

**現象：** ログインページに「ユーザー名またはパスワードが正しくありません」表示

**解決手順：**
1. 初回ログインはデフォルト管理者アカウント使用：ユーザー名`admin`、パスワード`admin123`
2. デフォルトパスワード無効の場合、管理者にパスワードリセット依頼
3. ユーザー存在確認

---

### セッション期限切れ：自動ログアウト

**現象：** 使用中ページが自動的にログインページに遷移

**原因：**
1. セッション有効期限経過（デフォルト24時間）
2. ブラウザCookie削除
3. サービス再起動によるセッション無効化

**解決手順：** 再ログインで復帰

---

### アカウント無効：ログイン不可

**現象：** ログイン失敗、「Account is disabled」表示

**解決手順：** 管理者にアカウント再有効化依頼：
```bash
docker compose exec postgres psql -U ace -d ace -c "UPDATE users SET is_active=true WHERE username='xxx';"
```

---

### 権限不足：管理機能にアクセスできない

**現象：** 管理ページアクセス時「Admin access required」表示

**解決手順：**
1. 現在のユーザー権限確認
2. 管理者にユーザー権限をadminに変更依頼

---

### パスワード変更失敗：パスワードが短すぎる

**現象：** パスワード変更時「New password must be at least 8 characters」表示

**解決手順：**
1. 新パスワード8文字以上確認
2. 新パスワードと現在のパスワードが異なること確認

---

## 三、ワークスペースとプロジェクト管理

### プロジェクト作成失敗：パス権限不足

**現象：** プロジェクト作成時「Permission denied to create directory」表示

**解決手順：**
1. ユーザーsystem_account権限確認：`sudo chown -R <user>:<group> /path`
2. 権限付与または別パス使用
3. マルチユーザーモードデフォルトパス：`/workspace/<username>/`

---

### プロジェクトパスが存在しないまたはアクセス不可

**現象：** プロジェクト開封時「Directory does not exist」表示

**解決手順：**
1. パス存在とディレクトリ確認
2. パス不存在の場合、プロジェクト再作成

---

### プロジェクト既存：重複作成

**現象：** プロジェクト作成時「Project already exists」表示

**解決手順：** 別パスで新プロジェクト作成、または既存プロジェクト削除後再作成

---

### マルチユーザーモードWorkspace起動失敗

**現象：** ワークスペース进入時「Failed to get user workspace URL」表示

**原因：**
1. qwen-code-webui未インストールまたはパス設定エラー
2. ユーザーsystem_accountシステムユーザー不存在
3. sudo設定問題

**解決手順：**
1. qwen-code-webui可用性確認：`which qwen-code-webui`
2. ユーザーsystem_account存在確認：`id <account>`
3. sudoers設定確認
4. 起動ログ確認：`tail -f /tmp/open-ace-*.log`

---

### Workspaceインスタンス数上限到達

**現象：** 新セッション作成時「Maximum instances (20) reached」表示

**解決手順：**
1. 空きインスタンス自動クリーン待機（デフォルト30分タイムアウト）
2. 管理者設定変更で上限増加（max_instances）

---

## 四、セッションとAI対話

### クォータ超過：Workspace無効

**現象：** ワークスペースにクォータ超過警告表示、AI機能使用不可

**原因：**
1. 日/月Token使用量がクォータ制限超過
2. 日/月リクエスト数がクォータ制限超過

**解決手順：**
1. DashboardページでUsage Overview確認
2. クォータリセット待機（日次クォータ毎日リセット、月次クォータ毎月リセット）
3. 管理者にクォータ調整依頼

---

### リモートマシンオフライン：リモートセッション作成不可

**現象：** リモートセッション作成時「Failed to create remote session」表示

**原因：**
1. リモートAgent未実行またはネットワーク不通
2. Agent登録失効
3. ユーザーがマシンに未割り当て

**解決手順：**
1. リモートマシン状態確認
2. Agentサービス実行確認：`systemctl status open-ace-agent`
3. Agent再登録
4. ユーザーがマシンに割り当て確認

---

### リクエストタイムアウトまたはネットワークエラー

**現象：** APIリクエスト失敗、「Request timed out」または「Network error」表示

**原因：**
1. ネットワーク接続不安定
2. サーバー応答遅延または高負荷
3. リクエストタイムアウト（デフォルト30秒）

**解決手順：**
1. ネットワーク接続状態確認
2. ページ更新で再試行（フロントエンド自動再試行3回）
3. サービス状態確認

---

### セッションが見つからない

**現象：** セッション詳細開封時「Session not found」表示

**原因：**
1. セッション削除または期限切れ
2. セッションIDエラー
3. ユーザーアクセス権限なし

**解決手順：**
1. セッションID正確性確認
2. セッションリストで有効セッション検索
3. リモートセッションの場合、リモートマシンオンライン確認

---

## 五、システム設定

### 言語切り替え

**解決手順：** ログインページまたは設定ページで言語選択。対応言語：
- English（英語）
- 中文（簡体字中国語）
- 日本語
- 한국어（韓国語）

---

### テーマ切り替え（暗色/明色）

**解決手順：** インターフェース上部または設定でテーマ切り替えボタンを見つけ、Light / Darkモード選択

---

### ページ更新後設定消失

**原因：** ブラウザがローカルストレージ無効化

**解決手順：** ブラウザがlocalStorage使用許可確認、設定再構成

---
