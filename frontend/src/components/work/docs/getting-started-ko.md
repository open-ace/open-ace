# 🚀 Getting Started — 빠른 시작 가이드

> Open ACE에 오신 것을 환영합니다! 이 가이드는 **처음 사용하는 분**을 위해 빠르게 시작하고 헤매지 않도록 만들어졌습니다.

---

## 1. Open ACE란 무엇인가?

**Open ACE**는 오픈소스 **기업용 AI 작업 플랫폼**으로, 두 가지 주요 문제를 해결합니다:

| 문제 | 해결책 |
|------|--------|
| 🤔 **AI를 효과적으로 사용하고 있는가?** | **Work 모드** — AI를 효율적으로 사용하여 생산성 향상 |
| 😰 **AI를 적절히 관리하고 있는가?** | **Manage 모드** — 관리자에게 AI 사용 현황의 완전한 제어권 제공, 리스크 감소 |

**주요 기능:**
- 🤖 **다중 AI 도구 통합** — Qwen Code, Claude Code, OpenClaw 등 주요 AI 도구 지원, 하나의 플랫폼에서 완료
- 💬 **스마트 대화 관리** — 기록 저장, 세션 복원, 컨텍스트 메모리, 대화가 중단되지 않음
- 📝 **프롬프트 라이브러리** — 팀 공유의 고품질 프롬프트, 베스트 프랙티스를 1클릭으로 재사용
- 🌐 **리모트 워크스페이스** — 브라우저에서 리모트 머신을 운영, AI가 리모트 서버에서 직접 실행
- 📊 **사용량 분석** — 토큰 소비, 비용 분석, 효율 개선, 데이터 기반 의사결정

**누구에게 적합한가?**
- AI 코딩 어시스턴트를 자주 사용하는 개발자
- AI 도구와 비용을 중앙에서 관리하려는 팀
- 프라이빗 배포로 데이터 보안을 확보하려는 조직

## 2. 첫 사용 전 준비

### 1. 기본 환경

시작 전, 다음을 확인하세요:

- ✅ **Open ACE가 배포됨** — 팀이 Open ACE 플랫폼을 배포함 (로컬 또는 리모트)
- ✅ **계정 등록됨** — 로그인 계정이 있음 (관리자에게 연락 또는 기본 계정 사용)
- ✅ **브라우저 지원** — Chrome, Edge, Firefox 등 모던 브라우저 추천

### 2. 플랫폼에 로그인

1. 브라우저를 열고 Open ACE 주소에 접속 (예: `http://localhost:5000` 또는 배포 주소)
2. **사용자명**과 **비밀번호**를 입력하고 로그인 클릭

> 💡 **기본 관리자 계정**: `admin` / `admin123` (프로덕션 환경에서는 기본 비밀번호를 변경하세요!)

### 3. 주요 설정 항목 (관리자 작업)

**관리자**인 경우, 첫 사용 전 설정이 필요:

#### API Key 추가

**왜 설정이 필요한가?** Open ACE 자체는 AI 모델을 제공하지 않으며, 여러 AI 도구의 API Key를 **중앙에서 관리**하여 팀원이 개별 설정할 필요가 없습니다.

**설정 방법:**
1. 로그인 후, **Manage 모드**로 전환 (상단 모드 스위처 클릭)
2. 왼쪽 사이드바 → **리모트 워크스페이스** → **API Keys**
3. **"API Key 추가"** 버튼 클릭
4. Provider 선택 (예: OpenAI, Anthropic, Google), API Key와 옵션 Base URL 입력
5. 저장 클릭

> ⚠️ **중요**: 리모트 워크스페이스 기능은 관리 페이지에서 API Key 추가 **필수**, 환경 변수의 Key는 자동으로 작동하지 않습니다.

## 3. 코어 기능 빠른 시작

### 시나리오 1: 새 대화 세션 생성

1. 로그인 후, 기본적으로 **Work 모드**에 진입
2. **왼쪽 사이드바**에서 **"Sessions"** 영역 찾기
3. **"새 세션"** 버튼 클릭 (일반적으로 `+` 아이콘)
4. 대화상자에서:
   - **워크스페이스 타입** 선택 (로컬 워크스페이스 또는 리모트 워크스페이스)
   - 리모트 워크스페이스 선택 시, 목록에서 **온라인 리모트 머신** 선택
   - **프로젝트 경로** 입력 (리모트 머신의 작업 디렉토리, 기본값 자동 입력)
   - **세션 제목** 입력 (옵션, 나중에 검색하기 쉽게)
5. **"생성"** 클릭, 새 워크스페이스 탭이 열림

### 시나리오 2: 다른 AI 모델 선택

1. 워크스페이스 페이지에서, 상단의 **도구 선택기** 찾기
2. 드롭다운 메뉴 클릭, 사용 가능한 AI 도구 표시:
   - **Qwen** — Qwen 시리즈 (예: qwen3-coder-plus)
   - **Claude** — Anthropic Claude 시리즈
   - **OpenClaw** — 다른 OpenAI 호환 모델
3. 도구 선택 후, 시스템이 자동으로 해당 모델 로드
4. 일부 도구는 **모델명 지정** 지원, 세션 생성 시 입력

> 💡 **힌트**: 사용 가능한 AI 도구는 관리자가 설정한 API Key에 따라 다릅니다. 도구가 작동하지 않으면 관리자에게 API Key가 유효한지 확인하세요.

### 시나리오 3: 프롬프트 템플릿 사용

**프롬프트 템플릿이란?** 미리 작성된 고품질 프롬프트로, 매번 처음부터 쓸 필요 없이 빠르게 시작할 수 있습니다.

**사용 방법:**

1. **오른쪽 사이드바**에서 **"Prompts"** 탭 클릭
2. 프롬프트 목록 표시, 원하는 프롬프트 클릭하여 상세 보기
3. 프롬프트 옆의 **복사 아이콘** 클릭하여 내용을 클립보드에 복사
4. 복사한 내용을 워크스페이스 입력 상자에 붙여넣고, 필요시 수정 후 전송

**더 많은 프롬프트 보기:**
1. 왼쪽 사이드바의 **"Prompts"** 네비게이션 항목 클릭
2. 프롬프트 관리 페이지로 진입, 모든 프롬프트를 보기, 검색, 필터링
3. 원하는 프롬프트 카드 클릭하여 전체 내용과 사용 설명 보기

## 4. 인터페이스 기능 설명

Open ACE는 두 가지 모드 제공: **Work 모드** (모든 사용자 사용 가능)와 **Manage 모드** (관리자만).

### 🔵 Work 모드 — 모든 사용자 접근 가능

Work 모드는 일반 사용자의 일상 작업 인터페이스, **3열 레이아웃** 사용:

```
┌─────────────────────────────────────────────────────────────┐
│  [Logo] Open ACE        [모드 스위처]        [사용자 정보]    │  ← 상단 네비게이션
├──────────┬──────────────────────────────┬──────────────────┤
│          │                              │                  │
│  세션    │                              │  프롬프트         │
│  목록    │      메인 워크스페이스        │  도구            │  ← 오른쪽 툴바
│          │   (대화/코드 작업)            │  문서            │
│          │                              │                  │
│  네비    │                              │                  │
│  메뉴    │                              │                  │
├──────────┴──────────────────────────────┴──────────────────┤
│  [상태 바: 토큰 사용량 | 리퀘스트 사용량]                     │  ← 하단 상태 바
└─────────────────────────────────────────────────────────────┘
   ↑ 왼쪽 사이드바          ↑ 중앙 메인 영역
```

#### 왼쪽 사이드바 네비게이션

| 네비게이션 항목 | 설명 |
|-----------------|------|
| **Workspace** | 기본 AI 대화 인터페이스, AI와 대화, 코드 작성 |
| **Sessions** | 모든 기록 세션 보기, 검색 및 필터링, 이전 대화 복원 |
| **Prompts** | 프롬프트 라이브러리 보기 및 관리, 고품질 프롬프트 빠른 재사용 |
| **My Usage** | 개인 토큰 사용 현황 및 트렌드 분석 보기 |
| **Insights** | AI 사용 효율 분석 및 개선 제안 받기 |

#### 중앙 메인 영역

- **대화 영역**: AI와의 대화 기록 표시, 메시지, 코드 블록, 생각 과정 포함
- **입력 상자**: 하단에서 질문 또는 명령 입력, Enter로 전송
- **툴바**: 입력 상자 위 또는 옆, 전송, 일시정지, 중지 버튼 제공
- **풀스크린 모드**: 풀스크린 버튼 클릭하여 사이드 패널 숨기고 대화에 집중 (ESC로 종료)

#### 오른쪽 툴바

| 탭 | 설명 |
|-----|------|
| **프롬프트** | 일반적인 프롬프트 빠르게 보기, 1클릭 복사 |
| **도구** | 다른 AI 도구 전환 빠른 엔트리 (Qwen, Claude, OpenClaw) |
| **문서** | 도움 문서 엔트리 (Getting Started, FAQ 등) |

> 💡 **힌트**: 오른쪽 패널은 접기/펼치기 가능, 패널 제목 옆의 화살표 클릭.

---

### 🟠 Manage 모드 — 관리자만 접근 가능

Manage 모드는 관리자 전용 백엔드 관리 인터페이스. **일반 사용자는 이 모드에 접근할 수 없습니다**, `/manage/*`에 접근하면 Work 모드로 리디렉션됩니다.

#### 네비게이션 그룹

| 그룹 | 네비게이션 항목 | 설명 |
|-------|-----------------|------|
| **개요** | Dashboard | 시스템 개요, 주요 지표 한눈에 확인 |
| **분석** | Token Trend | 토큰 소비 트렌드 분석 그래프 |
| | Request Statistics | API 리퀘스트 통계 및 시각화 |
| | Anomaly Detection | 비정상 사용 행동 감지 및 알림 |
| | ROI Analysis | 투자 수익률 분석 리포트 |
| | Conversation History | 모든 사용자의 대화 기록 |
| | Messages | 메시지 통계 및 관리 |
| **거버넌스** | Audit Center | 운영 감사 로그, 컴플라이언스 추적 |
| | Quota & Alerts | 사용자 할당 설정 및 알림 규칙 |
| | Compliance Management | 컴플라이언스 정책 설정 및 관리 |
| | Security Center | 보안 설정 및 리스크 모니터링 |
| **사용자** | User Management | 사용자 계정 관리, 권한 할당 |
| | Tenant Management | 멀티 테넌트 관리, 테넌트 분리 |
| **프로젝트** | Project Management | 프로젝트 생성 및 관리 |
| **리모트 워크스페이스** | Remote Machines | 리모트 머신 등록 및 관리 |
| | API Keys | API Key 중앙 관리 |
| **설정** | SSO Settings | 싱글 사인온 설정 |

> 💡 **힌트**: Manage 모드의 모든 기능은 관리자 전용, 일반 사용자는 접근할 수 없습니다.

## 5. FAQ — 자주 묻는 질문과 문제 해결

### Q1: 메시지를 전송했는데 응답이 없는가?

**가능한 원인과 해결책:**

1. **API Key가 미설정 또는 무효**
   - 관리자에게 API Key가 올바르게 설정되었는지 확인
   - API Key가 만료 또는 비활성화되었는지 확인

2. **네트워크 연결 문제**
   - 브라우저 네트워크가 정상인지 확인
   - 페이지 새로고침 또는 재로그인 시도

3. **AI 도구가 미선택**
   - 상단 도구 선택기에서 유효한 AI 도구가 선택되었는지 확인
   - 도구 목록이 비어있으면 백엔드에서 사용 가능한 API Key가 설정되지 않음

### Q2: 모델을 선택할 수 없거나 에러가 표시되는가?

**일반적인 상황:**

- **도구가 표시되지만 작동하지 않음** → API Key가 무효할 가능성, 관리자에게 확인
- **모델명 에러** → 모델명이 올바른지 확인 (예: `qwen3-coder-plus`)
- **"사용 가능한 API Key 없음" 프롬프트** → 관리자가 관리 페이지에서 해당 API Key를 추가 필요

### Q3: 다른 모드를 전환하는 방법?

Open ACE는 두 가지 모드가 있습니다:

| 모드 | 엔트리 | 대상 사용자 |
|------|--------|-------------|
| **Work 모드** | 기본 모드, 모든 사용자 사용 가능 | 일반 사용자, 개발자 |
| **Manage 모드** | 상단 모드 스위처 클릭 → "Manage" 선택 | 관리자만 표시 |

**전환 방법:**
1. 페이지 상단에서 **모드 스위처** 찾기 (일반적으로 "Work" 또는 "Manage" 표시)
2. 스위처 클릭, 원하는 모드 선택
3. 페이지가 자동으로 해당 모드의 홈페이지로 리디렉션

### Q4: 이전 대화를 찾는 방법?

1. 왼쪽 사이드바의 **"Sessions"** 클릭
2. **검색 상자**에서 키워드 입력, 기록 세션 필터링
3. 원하는 세션 카드 클릭하여 전체 대화 보기
4. 해당 세션을 계속하려면 **"세션 복원"** 버튼 클릭

### Q5: 리모트 워크스페이스에서 머신이 보이지 않는가?

**문제 해결 단계:**

1. **머신이 등록되었는지 확인** → 관리자에게 리모트 머신이 등록되고 온라인인지 확인
2. **권한이 할당되었는지 확인** → 관리자가 관리 페이지에서 머신을 당신에게 할당 필요
3. **네트워크 연결성 확인** → 리모트 머신이 Open ACE 서버에 접근 가능한지 확인
4. **머신 상태 확인** → 머신이 "오프라인"을 표시하면 리모트 Agent가 정상 작동하는지 확인

### Q6: 프롬프트를 복사하는 방법?

1. 오른쪽 패널의 "프롬프트" 탭에서, 원하는 프롬프트 찾기
2. 프롬프트 오른쪽의 **복사 아이콘** (클립보드 아이콘) 클릭
3. 시스템이 "복사됨"을 표시, 워크스페이스 입력 상자에 붙여넣기

### Q7: 토큰 사용량은 어디서 확인?

- **개인 사용량**: 왼쪽 사이드바 → **My Usage**
- **상세 리포트**: 왼쪽 사이드바 → **Insights**
- **실시간 사용량**: 하단 상태 바가 현재 세션의 토큰 소비를 표시

### Q8: 풀스크린 모드를 종료하는 방법?

- 키보드의 **ESC** 키 누르기
- 또는 풀스크린 버튼 클릭 (표시되는 경우)

- 베스트 프랙티스를 팀과 공유

### 🌐 리모트 워크스페이스 고급 기능

- 리모트 머신 등록 방법, Agent 설치 학습
- 다른 시나리오용 여러 AI 도구 API Key 설정
- 일시정지/복원 기능을 사용하여 장시간 실행 작업 관리

### 📊 데이터 분석 (관리자)

- Manage 모드로 전환, 팀 전체 사용량 보기
- 할당 알림 설정, 과다 소비 방지
