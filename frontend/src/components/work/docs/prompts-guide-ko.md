# 프롬프트 가이드

---

## 소개

**Open ACE** 프롬프트 사용 가이드에 오신 것을 환영합니다! 이 가이드는 효과적인 프롬프트 작성 기법을 통해 Open ACE 플랫폼의 AI 기능을 최대한 활용하고 작업 효율을 크게 향상시키는 데 도움을 드립니다.

Open ACE는 Claude Code, Qwen Code 등 주요 AI 도구를 지원하는 기업급 AI 작업 플랫폼입니다:
- 🚀 AI와 효율적으로 대화하여 다양한 작업 완료
- 📝 프롬프트 라이브러리를 사용하여 팀의 베스트 프랙티스 재사용
- 📊 사용 데이터를 추적하여 워크플로우 최적화
- 🔒 안전하고 규정 준수하여 AI 사용, 기업 데이터 보호

## 프롬프트란

**프롬프트(Prompt)**는 AI에 보내는 지시 또는 질문입니다. 간단히 말해서, 프롬프트는 AI와의 통신 언어입니다.

### 프롬프트 예시

```
다음 주 월요일 프로젝트 진행 보고 회의에 대한 초대 이메일을 작성해 주세요.
```

좋은 프롬프트는:
- ✅ AI가 당신의 요구를 정확하게 이해
- ✅ 고품질 답변을 얻음
- ✅ 통신 시간을 절약하고 효율 향상

## 프롬프트 기초

### 1. 프롬프트 기본 구조

완전한 프롬프트는 일반적으로 다음 요소를 포함합니다:

| 요소 | 설명 | 예시 |
|------|------|------|
| **역할 설정** | AI에게 역할을 지시 | "당신은 경험 많은 제품 관리자입니다" |
| **작업 설명** | 완료할 작업을 명확히 | "제품 요구 사항 문서를 작성해 주세요" |
| **배경 정보** | 필요한 컨텍스트 제공 | "이것은 중소기업용 협업 도구입니다" |
| **출력 요구** | 기대하는 출력 형식 지정 | "Markdown 형식, 사용자 스토리와 수용 기준 포함" |

### 2. 프롬프트 유형

#### 대화형 프롬프트
일상적인 질문과 인터랙션용:

```
애자일 개발이란 무엇인가요? 간단히 설명해 주세요.
```

#### 작업형 프롬프트
특정 작업 완료용:

```
다음 영어 이메일을 중국어로 번역하고, 비즈니스 어조를 유지해 주세요:

Dear Team,
I would like to schedule a meeting...
```

#### 창의형 프롬프트
창의적 콘텐츠 생성용:

```
젊은층을 대상으로 한 스포츠 드링크의 브랜드 슬로건을 디자인해 주세요:
- 짧고 강력하게, 8글자 이내
- 활기와 스포츠 정신을 반영
- 쉽게 기억할 수 있게
```

## Open ACE 플랫폼 특징

### 📝 프롬프트 라이브러리

Open ACE는 기업급 프롬프트 라이브러리 기능을 제공:
- 일반적인 프롬프트 템플릿 저장
- 우수한 프롬프트를 팀원과 공유
- 베스트 프랙티스를 한 번 클릭으로 재사용

**사용 방법:**
1. 왼쪽 메뉴의 "프롬프트 라이브러리" 클릭
2. "새 프롬프트" 클릭
3. 제목, 설명, 프롬프트 내용 입력
4. 공유 범위 선택 (개인/팀/전사)

### 💬 세션 관리

Open ACE의 스마트 세션 관리 기능:
- 대화 역사 자동 저장
- 세션 복원, 컨텍스트 유지
- 역사 대화 빠른 검색

### 🔍 스마트 검색

세션 간 역사 대화 내용 검색, 지식 축적:
- 키워드로 검색
- 시간 범위로 필터
- 도구 유형으로 필터

## 효과적인 프롬프트 작성 기법

### 기법 1: 역할 설정 명확화

❌ 나쁜 예시:
```
제품 설명을 작성해 줘
```

✅ 좋은 예시:
```
당신은 10년 경력의 기술 문서 작성 전문가입니다.
새로 개발한 CRM 시스템의 제품 매뉴얼을 작성해 주세요.
대상 사용자는 영업 팀, 다음 섹션을 포함:
1. 제품 개요
2. 핵심 기능 소개
3. 빠른 시작 가이드
4. FAQ

언어는 간결하고 전문적으로, 과도한 기술 용어를 피하세요.
```

### 기법 2: 충분한 배경 제공

❌ 나쁜 예시:
```
매출 데이터를 분석해 줘
```

✅ 좋은 예시:
```
2024년 1분기 매출 데이터를 분석하고 싶습니다.

배경 정보:
- 제품: 기업급 SaaS 소프트웨어
- 타겟 마켓: 아시아태평양 중소기업
- 데이터 범위: 2024년 1월-3월
- 분석 목적: 시장 확장 전략 효과 평가

다음 차원에서 분석해 주세요:
1. 매출 추세
2. 고객 확보 비용
3. 지역 분포
4. 제품 라인 기여도

출력 형식: 표와 차트로 주요 데이터를 표시.
```

### 기법 3: 구조화된 출력 요구

특정 형식이 필요할 때, 명확히 지정:

```
다음 JSON 형식으로 제품 정보를 출력해 주세요:

{
  "product_name": "제품명",
  "features": ["기능1", "기능2"],
  "target_users": "타겟 사용자",
  "pricing": "가격 전략"
}

온라인 교육 플랫폼의 제품 정보를 생성해 주세요.
```

### 기법 4: 단계별 가이드

복잡한 작업, AI를 단계별로 가이드:

```
새 제품의 마케팅 계획을 수립해 주세요:

단계 1: 타겟 사용자 페르소나 분석
단계 2: 핵심 판매 포인트 확정
단계 3: 마케팅 채널 선택
단계 4: 콘텐츠 전략 수립
단계 5: KPI 지표 설정

먼저 단계 1을 완료하고, 확인 후 계속해 주세요.
```

### 기법 5: 예시로 가이드

예시로 AI가 요구를 더 잘 이해:

```
다음 스로 제품 홍보 문안을 작성해 주세요:

예시:
「복잡함을 벗어나고, 효율을 embrace —— XX 협업 플랫폼」

스마트 고객 서비스 시스템의 홍보 문안을 작성해 주세요.
제품 특징:
- 24시간 365일 온라인
- 다국어 지원
- 스마트 학습 최적화
```

## 시나리오별 프롬프트 템플릿

### 📧 이메일 작성

```
당신은 전문적인 비즈니스 이메일 작성 어시스턴트입니다.
{이메일 유형} 이메일을 작성해 주세요.

수신자: {수신자 역할}
주제: {이메일 주제}
핵심 내용: {전달할 핵심 정보}
기대 어조: {정식/친절/긴급}

요구: 언어는 간결하고 명확하게, 주요 정보를 강조.
```

### 📊 데이터 분석

```
당신은 데이터 분석 전문가입니다. 다음 데이터를 분석해 주세요:

데이터 내용: {데이터를 붙여넣기 또는 데이터 소스 설명}
분석 목적: {어떤 결론을 얻고 싶은지}
분석 차원: {주목할 지표}

제공해 주세요:
1. 데이터 개요
2. 주요 발견
3. 추세 분석
4. 제안 조치
```

### 📝 문서 작성

```
당신은 기술 문서 작성 전문가입니다.
{문서 유형}을 작성해 주세요.

프로젝트 배경: {프로젝트 설명}
타겟 독자: {문서 수용자}
문서 목적: {달성하고 싶은 목표}

요구:
- 구조 명확, Markdown 형식 사용
- 필요한 예시와 차트 설명 포함
- 전문 용어는 설명
```

### 💡 창의 생성

```
당신은 창의 계획 전문가입니다.
아이디어 브레인스토밍을 해주세요.

프로젝트/제품: {프로젝트 설명}
창의 방향: {창의가 필요한 영역}
제약 조건: {제한 요소}

제공해 주세요:
1. 최소 5개 창의 방향
2. 각 방향의 실현 가능성 분석
3. 추천 방안과 이유
```

### 🐛 문제 진단

```
당신은 기술 지원 전문가입니다.
다음 문제를 진단해 주세요:

문제 설명: {문제 현상}
환경 정보: {시스템 버전, 소프트웨어 버전 등}
시도한 해결 방안: {이미 시도한 것}
오류 정보: {특정 오류 메시지}

제공해 주세요:
1. 문제 원인 분석
2. 해결 단계
3. 예방 조치
```

## 베스트 프랙티스

### ✅ DO - 추천 사항

1. **명확하고 구체적**: 요구를 가능한 구체적으로 표현
2. **컨텍스트 제공**: 충분한 배경 정보를 제공
3. **반복 최적화**: AI 답변에 따라 프롬프트를 조정
4. **템플릿 저장**: 효과적인 프롬프트를 라이브러리에 저장
5. **프라이버시 보호**: 프롬프트에 민감 정보를 포함하지 않음

### ❌ DON'T - 피해야 할 사항

1. **모호함**: "무엇을 작성해" 등 모호한 지시를 피함
2. **정보 과다**: 한 번에 과도한 정보를 제공
3. **컨텍스트 무시**: 여러 차례 대화에서 이전 교류를 무시
4. **과도한 기대**: AI의 한계를 이해
5. **기밀 누설**: 기업 AI 플랫폼 외에서 민감 정보를 공유하지 않음

## 자주 묻는 질문

### Q1: AI의 답변이 기대와 다른 이유?

**가능한 원인:**
- 프롬프트가 명확하고 구체적이지 않음
- 필요한 배경 정보 부족
- 출력 요구가 구체적이지 않음

**해결 방법:**
- 프롬프트를 재구성하고, 구체 요구를 추가
- 기대하는 출력 형식의 예시를 제공
- AI를 단계별로 가이드

### Q2: 프롬프트 라이브러리의 템플릿 사용 방법?

1. "프롬프트 라이브러리"를 열기
2. 필요한 템플릿을 찾기
3. "사용" 버튼을 클릭
4. 실제 상황에 따라 플레이스홀더 내용을 수정
5. AI에 전송

### Q3: 프롬프트를 팀원과 공유하는 방법?

1. 프롬프트를 생성 또는 편집
2. "이 프로럼트를 공개" 체크
3. 저장을 클릭

공개된 프롬프트는 다른 팀원이 조회 및 사용할 수 있습니다.

### Q4: AI 답변이 너무 길/짧은 경우?

프롬프트에서 명확히 지정:
- 짧은 답변이 필요: "3문장 이내로 답변해 주세요"
- 상세 답변이 필요: "상세히 설명하고, 구체 단계와 예시를 포함"
