# 🚀 Getting Started — クイックスタートガイド

## 1. Open ACEとは？

**Open ACE**はオープンソースの**企業向けAIワークプラットフォーム**で、2つの主要な問題を解決します：

| 問題 | 解決策 |
|------|--------|
| 🤔 **AIを効果的に使っているか？** | **Workモード** — AIを効率的に使用して生産性を向上 |
| 😰 **AIを適切に管理しているか？** | **Manageモード** — 管理者にAI使用状況の完全な制御を提供、リスクを低減 |

**主な機能：**
- 🤖 **複数AIツール統合** — Qwen Code、Claude Code、OpenClawなどの主要AIツールをサポート、1つのプラットフォームで完結
- 💬 **スマート会話管理** — 履歴記録、セッション復元、コンテキスト記憶、会話が中断されない
- 📝 **プロンプトライブラリ** — チーム共有の高品質プロンプト、ベストプラクティスを1クリックで再利用
- 🌐 **リモートワークスペース** — ブラウザでリモートマシンを操作、AIがリモートサーバーで直接実行
- 📊 **使用量分析** — トークン消費、コスト分析、効率改善、データ駆動の意思決定

**誰に適しているか？**
- AIコーディングアシスタントを頻繁に使用する開発者
- AIツールとコストを一元管理したいチーム
- プライベートデプロイでデータセキュリティを確保したい組織

## 2. 初回使用前の準備

### 1. 基本環境

開始前に、以下を確認してください：

- ✅ **Open ACEがデプロイ済み** — チームがOpen ACEプラットフォームをデプロイしている（ローカルまたはリモート）
- ✅ **アカウント登録済み** — ログインアカウントがある（管理者に連絡またはデフォルトアカウント使用）
- ✅ **ブラウザサポート** — Chrome、Edge、Firefoxなどのモダンブラウザ推奨

### 2. プラットフォームにログイン

1. ブラウザを開き、Open ACEアドレスにアクセス（例：`http://localhost:5000` またはデプロイアドレス）
2. **ユーザー名**と**パスワード**を入力し、ログインをクリック

> 💡 **デフォルト管理者アカウント**: `admin` / `admin123` （本番環境では必ずデフォルトパスワードを変更してください！）

### 3. 重要設定項目（管理者操作）

**管理者**の場合、初回使用前に設定が必要：

#### API Keyの追加

**なぜ設定が必要？** Open ACE自体はAIモデルを提供せず、複数AIツールのAPI Keyを**一元管理**し、チームメンバーが個別に設定不要になります。

**設定手順：**
1. ログイン後、**Manageモード**に切り替え（上部モードスイッチャーをクリック）
2. 左サイドバー → **リモートワークスペース** → **API Keys**
3. **"API Keyを追加"** ボタンをクリック
4. Providerを選択（例：OpenAI、Anthropic、Google）、API KeyとオプションのBase URLを入力
5. 保存をクリック

> ⚠️ **重要**: リモートワークスペース機能は管理ページでAPI Keyを追加**必須**、環境変数のKeyは自動的に有効になりません。

## 3. コア機能のクイックスタート

### シナリオ1：新しい会話セッションを作成

1. ログイン後、デフォルトで**Workモード**に入ります
2. **左サイドバー**で**"Sessions"**エリアを見つける
3. **"新規セッション"** ボタンをクリック（通常 `+` アイコン）
4. ダイアログで：
   - **ワークスペースタイプ**を選択（ローカルワークスペースまたはリモートワークスペース）
   - リモートワークスペースを選択した場合、リストから**オンラインのリモートマシン**を選択
   - **プロジェクトパス**を入力（リモートマシン上の作業ディレクトリ、デフォルトで自動入力）
   - **セッションタイトル**を入力（オプション、後で検索しやすくするため）
5. **"作成"** をクリック、新しいワークスペースタブが開きます

### シナリオ2：異なるAIモデルを選択

1. ワークスペースページで、上部の**ツールセレクター**を見つける
2. ドロップダウンメニューをクリック、利用可能なAIツールが表示：
   - **Qwen** — Qwenシリーズ（例：qwen3-coder-plus）
   - **Claude** — Anthropic Claudeシリーズ
   - **OpenClaw** — 他のOpenAI互換モデル
3. ツールを選択後、システムが自動的に対応するモデルをロード
4.一部のツールは**モデル名の指定**をサポート、セッション作成時に入力

> 💡 **ヒント**: 利用可能なAIツールは管理者が設定したAPI Keyに依存します。ツールが動作しない場合、管理者にAPI Keyが有効か確認してください。

### シナリオ3：プロンプトテンプレートを使用

**プロンプトテンプレートとは？** 事前に書かれた高品質プロンプトで、毎回ゼロから書く必要なく、すぐに始められます。

**使用方法：**

1. **右サイドバー**で**"Prompts"** タブをクリック
2. プロンプトリストが表示、任意のプロンプトをクリックして詳細を表示
3. プロンプトの横の**コピーアイコン**をクリックして内容をクリップボードにコピー
4. コピーした内容をワークスペースの入力ボックスに貼り付け、必要に応じて修正して送信

**更多プロンプトを閲覧：**
1. 左サイドバーの**"Prompts"**ナビゲーション項目をクリック
2. プロンプト管理ページに入り、すべてのプロンプトを閲覧、検索、フィルタリング
3. 任意のプロンプトカードをクリックして、完全な内容と使用説明を表示

## 4. インターフェース機能説明

Open ACEは2つのモードを提供：**Workモード**（すべてのユーザー利用可能）と**Manageモード**（管理者のみ）。

### 🔵 Workモード — すべてのユーザーがアクセス可能

Workモードは一般ユーザーの日常作業インターフェース、**3列レイアウト**を使用：

```
┌─────────────────────────────────────────────────────────────┐
│  [Logo] Open ACE        [モードスイッチャー]    [ユーザー情報] │  ← 上部ナビゲーション
├──────────┬──────────────────────────────┬──────────────────┤
│          │                              │                  │
│  セッション│                              │  プロンプト       │
│  リスト   │      メインワークスペース      │  ツール          │  ← 右ツールバー
│          │   （会話/コード操作）          │  ドキュメント     │
│          │                              │                  │
│  ナビ     │                              │                  │
│  メニュー │                              │                  │
├──────────┴──────────────────────────────┴──────────────────┤
│  [ステータスバー: トークン使用量 | リクエスト使用量]            │  ← 下部ステータスバー
└─────────────────────────────────────────────────────────────┘
   ↑ 左サイドバー          ↑ 中央メインエリア
```

#### 左サイドバーナビゲーション

| ナビゲーション項目 | 説明 |
|-------------------|------|
| **Workspace** | デフォルトのAI会話インターフェース、AIと対話、コードを書く |
| **Sessions** | すべての履歴セッションを表示、検索とフィルタリング、以前の会話を復元 |
| **Prompts** | プロンプトライブラリを閲覧・管理、高品質プロンプトを迅速再利用 |
| **My Usage** | 個人のトークン使用状況とトレンド分析を表示 |
| **Insights** | AI使用効率分析と改善提案を取得 |

#### 中央メインエリア

- **会話エリア**: AIとの会話履歴を表示、メッセージ、コードブロック、思考プロセスを含む
- **入力ボックス**: 下部で質問やコマンドを入力、Enterで送信
- **ツールバー**: 入力ボックスの上または横、送信、一時停止、停止ボタンを提供
- **フルスクリーンモード**: フルスクリーンボタンをクリックしてサイドパネルを非表示、会話に集中（ESCで退出）

#### 右ツールバー

| タブ | 説明 |
|------|------|
| **プロンプト** | 一般的なプロンプトを迅速閲覧、1クリックでコピー |
| **ツール** | 異なるAIツールを切り替えるクイックエントリー（Qwen、Claude、OpenClaw） |
| **ドキュメント** | ヘルプドキュメントエントリー（Getting Started、FAQなど） |

> 💡 **ヒント**: 右パネルは折りたたみ/展開可能、パネルタイトルの横の矢印をクリック。

---

### 🟠 Manageモード — 管理者のみアクセス可能

Manageモードは管理者専用のバックエンド管理インターフェース。**一般ユーザーはこのモードにアクセスできません**、`/manage/*`にアクセスするとWorkモードにリダイレクトされます。

| | Request Statistics | APIリクエスト統計とビジュアライゼーション |

| | Conversation History | すべてのユーザーの会話履歴記録 |
| | Messages | メッセージ統計と管理 |
| **ガバナンス** | Audit Center | 操作監査ログ、コンプライアンス追跡 |
| | Quota & Alerts | ユーザー割当設定とアラートルール |
| | Compliance Management | コンプライアンスポリシー設定と管理 |
| | Security Center | セキュリティ設定とリスク監視 |
| **ユーザー** | User Management | ユーザーアカウント管理、権限割当 |

> 💡 **ヒント**: Manageモードのすべての機能は管理者専用、一般ユーザーはアクセスできません。

## 5. FAQ — よくある質問とトラブルシューティング

### Q1: メッセージを送信しても応答がない？

**可能な原因と解決策：**

1. **API Keyが未設定または無効**
   - 管理者にAPI Keyが正しく設定されているか確認
   - API Keyが期限切れまたは無効化されているか確認

2. **ネットワーク接続問題**
   - ブラウザのネットワークが正常か確認
   - ページを更新または再ログインを試行

3. **AIツールが未選択**
   - 上部ツールセレクターで有効なAIツールが選択されているか確認
   - ツールリストが空の場合、バックエンドで利用可能なAPI Keyが設定されていない

### Q2: モデルを選択できないまたはエラーが表示？

**よくある状況：**

- **ツールが表示されるが動作しない** → API Keyが無効の可能性、管理者に確認
- **モデル名エラー** → モデル名が正しいか確認（例：`qwen3-coder-plus`）
- **"利用可能なAPI Keyなし"プロンプト** → 管理者が管理ページで対応するAPI Keyを追加必要

### Q3: 異なるモードを切り替える方法？

Open ACEは2つのモードがあります：

| モード | エントリー | 対象ユーザー |
|--------|-----------|--------------|
| **Workモード** | デフォルトモード、すべてのユーザー利用可能 | 一般ユーザー、開発者 |
| **Manageモード** | 上部モードスイッチャーをクリック → "Manage"を選択 | 管理者のみ表示 |

**切り替え方法：**
1. ページ上部で**モードスイッチャー**を見つける（通常"Work"または"Manage"を表示）
2. スイッチャーをクリック、希望のモードを選択
3. ページが自動的に対応するモードのホームページにリダイレクト

### Q4: 以前の会話を見つける方法？

1. 左サイドバーの**"Sessions"**をクリック
2. **検索ボックス**でキーワードを入力、履歴セッションをフィルタリング
3. 任意のセッションカードをクリックして完全な会話を表示
4. そのセッションを続ける場合、**"セッションを復元"** ボタンをクリック

### Q5: リモートワークスペースでマシンが見えない？

**トラブルシューティング手順：**

1. **マシンが登録されているか確認** → 管理者にリモートマシンが登録されオンラインか確認
2. **権限が割り当てられているか確認** → 管理者が管理ページでマシンをあなたに割り当て必要
3. **ネットワーク接続性を確認** → リモートマシンがOpen ACEサーバーにアクセス可能か確認
4. **マシンステータスを確認** → マシンが"オフライン"を表示する場合、リモートAgentが正常に動作しているか確認

### Q6: プロンプトをコピーする方法？

1. 右パネルの"プロンプト"タブで、希望のプロンプトを見つける
2. プロンプトの右側の**コピーアイコン**（クリップボードアイコン）をクリック
3. システムが"コピー済み"を表示、ワークスペースの入力ボックスに貼り付け

### Q7: トークン使用量はどこで確認？

- **個人使用量**: 左サイドバー → **My Usage**
- **詳細レポート**: 左サイドバー → **Insights**
- **リアルタイム使用量**: 下部ステータスバーが現在のセッションのトークン消費を表示

### Q8: フルスクリーンモードを退出する方法？

- キーボードの**ESC**キーを押す
- またはフルスクリーンボタンをクリック（表示されている場合）
