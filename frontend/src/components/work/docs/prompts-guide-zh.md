## 什么是提示词

**提示词（Prompt）** 是您向 AI 发送的指令或问题。简单来说，提示词就是您与 AI 沟通的语言。

### 提示词示例

```
请帮我写一封会议邀请邮件，主题是下周一的项目进度汇报。
```

一个好的提示词能够：
- ✅ 让 AI 准确理解您的需求
- ✅ 获得高质量的回答
- ✅ 节省沟通时间，提高效率

## 提示词基础

### 1. 提示词的基本结构

一个完整的提示词通常包含以下要素：

| 要素 | 说明 | 示例 |
|------|------|------|
| **角色设定** | 告诉 AI 扮演什么角色 | "你是一位资深产品经理" |
| **任务描述** | 明确要完成什么任务 | "请帮我撰写产品需求文档" |
| **背景信息** | 提供必要的上下文 | "这是一个面向中小企业的协作工具" |
| **输出要求** | 指定期望的输出格式 | "使用 Markdown 格式，包含用户故事和验收标准" |

### 2. 提示词的类型

#### 对话型提示词
用于日常问答和交互：

```
什么是敏捷开发？请用简单的话解释。
```

#### 任务型提示词
用于完成具体工作：

```
请将以下英文邮件翻译成中文，保持商务语气：

Dear Team,
I would like to schedule a meeting...
```

#### 创意型提示词
用于生成创意内容：

```
请为一款面向年轻人的运动饮料设计品牌口号，要求：
- 简短有力，不超过8个字
- 体现活力与运动精神
- 容易记忆
```

## 高效提示词编写技巧

### 技巧一：明确角色设定

❌ 不好的示例：
```
帮我写个产品说明
```

✅ 好的示例：
```
你是一位有10年经验的技术文档撰写专家。请帮我为新开发的 CRM 系统
撰写产品说明书，目标用户是销售团队，需要包含以下部分：
1. 产品概述
2. 核心功能介绍
3. 快速上手指南
4. 常见问题解答

要求语言简洁专业，避免过多技术术语。
```

### 技巧二：提供充足背景

❌ 不好的示例：
```
分析一下销售数据
```

✅ 好的示例：
```
我需要分析2024年Q1的销售数据。

背景信息：
- 产品：企业级 SaaS 软件
- 目标市场：亚太区中小企业
- 数据范围：2024年1月-3月
- 分析目的：评估市场拓展策略效果

请从以下维度进行分析：
1. 销售额趋势
2. 客户获取成本
3. 区域分布情况
4. 产品线贡献度

输出格式：请使用表格和图表展示关键数据。
```

### 技巧三：结构化输出要求

当需要特定格式时，明确说明：

```
请按照以下 JSON 格式输出产品信息：

{
  "product_name": "产品名称",
  "features": ["特性1", "特性2"],
  "target_users": "目标用户",
  "pricing": "定价策略"
}

请为我们的在线教育平台生成产品信息。
```

### 技巧四：分步骤引导

对于复杂任务，分步骤引导 AI：

```
请帮我制定一个新产品的营销方案，分以下步骤进行：

第一步：分析目标用户画像
第二步：确定核心卖点
第三步：选择营销渠道
第四步：制定内容策略
第五步：设定 KPI 指标

请先完成第一步，等我确认后再继续。
```

### 技巧五：使用示例引导

通过示例让 AI 更好理解需求：

```
请按照以下风格撰写产品宣传文案：

示例：
「告别繁琐，拥抱高效 —— XX 协作平台，让团队协作如呼吸般自然」

请为我们的智能客服系统撰写宣传文案，产品特点是：
- 7x24小时在线
- 多语言支持
- 智能学习优化
```

## 场景化提示词模板

### 📧 邮件撰写

```
你是一位专业的商务邮件撰写助手。请帮我撰写一封{邮件类型}邮件。

收件人：{收件人角色}
主题：{邮件主题}
关键内容：{需要传达的核心信息}
期望语气：{正式/亲切/紧急}

要求：语言简洁明了，突出重点信息。
```

### 📊 数据分析

```
你是一位数据分析专家。请帮我分析以下数据：

数据内容：{粘贴数据或描述数据来源}
分析目的：{想要得出什么结论}
分析维度：{需要关注的指标}

请提供：
1. 数据概览
2. 关键发现
3. 趋势分析
4. 建议措施
```

### 📝 文档撰写

```
你是一位技术文档撰写专家。请帮我撰写{文档类型}。

项目背景：{项目描述}
目标读者：{文档受众}
文档目的：{希望达成的目标}

要求：
- 结构清晰，使用 Markdown 格式
- 包含必要的示例和图表描述
- 专业术语需要解释
```

### 💡 创意生成

```
你是一位创意策划专家。请帮我进行创意头脑风暴。

项目/产品：{项目描述}
创意方向：{需要创意的领域}
约束条件：{限制因素}

请提供：
1. 至少5个创意方向
2. 每个方向的可行性分析
3. 推荐方案及理由
```

### 🐛 问题诊断

```
你是一位技术支持专家。请帮我诊断以下问题：

问题描述：{问题现象}
环境信息：{系统版本、软件版本等}
已尝试的解决方案：{已经做过的尝试}
错误信息：{具体的错误提示}

请提供：
1. 问题原因分析
2. 解决步骤
3. 预防措施
```

## 最佳实践

### ✅ DO - 推荐做法

1. **清晰明确**：表达需求时尽可能具体
2. **提供上下文**：给出足够的背景信息
3. **迭代优化**：根据 AI 回复调整提示词
4. **保存模板**：将有效的提示词保存到提示词库
5. **保护隐私**：不要在提示词中包含敏感信息

### ❌ DON'T - 避免做法

1. **模糊不清**：避免"随便写点"这类模糊指令
2. **信息过载**：一次性提供过多信息
3. **忽视上下文**：在多轮对话中忽略之前的交流
4. **期望过高**：理解 AI 的局限性
5. **泄露机密**：避免在企业 AI 平台外分享敏感信息

## 常见问题

### Q1: 为什么 AI 的回答不是我想要的？

**可能原因：**
- 提示词不够清晰明确
- 缺少必要的背景信息
- 输出要求不够具体

**解决方法：**
- 重新组织提示词，增加具体要求
- 提供示例说明期望的输出格式
- 分步骤引导 AI 完成任务

### Q2: 提示词库中的模板如何使用？

1. 打开「提示词库」
2. 找到需要的模板
3. 点击「使用」按钮
4. 根据实际情况修改占位符内容
5. 发送给 AI

### Q3: 如何分享提示词给团队成员？

1. 创建或编辑提示词
2. 勾选「公开此提示词」（Make this prompt public）
3. 点击保存

公开后的提示词可被其他团队成员查看和使用。

### Q4: AI 回复太长/太短怎么办？

在提示词中明确说明：
- 需要简洁回答：「请用3句话以内回答」
- 需要详细回答：「请详细展开，包含具体步骤和示例」
