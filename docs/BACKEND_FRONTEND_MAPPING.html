2.1 Alerts (告警管理) - P1

后端 API 端点:

端点方法功能
/api/alertsGET获取告警列表
/api/alerts/unread-countGET获取未读告警数
/api/alerts/<alert_id>/readPOST标记告警已读
/api/alerts/read-allPOST标记全部已读
/api/alerts/<alert_id>DELETE删除告警
/api/alerts/preferencesGET/PUT获取/设置通知偏好
/api/alerts/streamGETSSE 实时告警流

建议前端页面:

  • 页面名称: AlertManagement (告警管理)
  • 路由: /manage/alerts
  • 所属分组: 治理

页面元素:

区域元素类型
统计卡片总告警数 / 未读告警 / 高危告警StatCard
过滤器告警类型 / 严重程度 / 已读状态Select
列表告警列表Table
设置通知偏好（邮件/推送/Webhook）Modal

2.2 Compliance (合规报告) - P1

后端 API 端点:

端点方法功能
/api/compliance/reportsGET/POST报告类型列表 / 生成报告
/api/compliance/reports/savedGET已保存报告
/api/compliance/audit/patternsGET审计模式分析
/api/compliance/audit/anomaliesGET审计异常检测
/api/compliance/audit/security-scoreGET安全评分
/api/compliance/retention/rulesGET/PUT数据保留规则
/api/compliance/retention/cleanupPOST执行数据清理
/api/compliance/retention/storageGET存储估算

建议前端页面:

2.2.1 合规报告页面

  • 路由: /manage/compliance/reports
  • 功能: 报告类型选择、日期范围、生成/下载报告

2.2.2 数据保留页面

  • 路由: /manage/compliance/retention
  • 功能: 保留规则管理、清理执行、存储估算

2.2.3 审计分析页面

  • 路由: /manage/compliance/audit
  • 功能: 模式分析、异常检测、用户画像、安全评分
